# v0.2.4 마이그레이션 가이드

시트에 L열 ID 신설 + GAS 4개 업데이트 + 위젯 빌드 작업 순서.
**디자인팀 휴무(연휴) 중에만 진행**. 디자이너가 시트를 만지지 않는 시간대에.

---

## 사전 체크

- [ ] 디자인팀 6명 모두 위젯 종료 상태 (또는 휴무 확인)
- [ ] 시트 백업 (메뉴 → 사본 만들기) — 마이그레이션 사고 대비
- [ ] 맥북에 git pull 완료 (최신 코드)

---

## 작업 순서

### STEP 1. 시트에 빈 컬럼 추가 (5분)

**💛신규·유지보수 시트**:
1. 시트에서 L열 헤더 우클릭 → "왼쪽에 열 1개 삽입"
2. 결과 확인: 기존 L(공유) → M, M(날짜 시작) → N. K열 옆에 빈 L열 생김

**💚완료 시트**:
1. 동일하게 L열 헤더 우클릭 → "왼쪽에 열 1개 삽입"
2. 결과: 기존 L(공유)→M, M(백업)→N, N(요청일)→O, O(공유일)→P, P(TAT)→Q

**업무 데이터 시트 (IMC 3본부 디자인팀 업무 데이터)**:
1. R열에 새 컬럼 추가 (이미 Q열까지 사용 중이라 R열은 빈 새 컬럼)
2. R1 셀(헤더 행)에 **`ID`** 입력
3. Q1 셀의 헤더가 `소요일`이면 → **`TAT`로 변경** (대시보드 API가 헤더 이름으로 찾음)

---

### STEP 2. GAS 코드 4개 업데이트 (20분)

각 GAS 프로젝트의 .gs 파일을 통째 새 코드로 교체.

#### 2-1. 디자인팀 스케줄러 자동화 프로젝트 (3개 파일)

GAS 콘솔 열기 → 해당 프로젝트 → 각 파일 내용 전체 선택 후 새 코드 붙여넣기:

| 파일명 | 새 코드 위치 |
|---|---|
| Code (또는 Scheduler) | `legacy-gas/Scheduler.gs` |
| Synccompletedtodatasheet | `legacy-gas/Synccompletedtodatasheet.gs` |
| schedule-widget-api | `schedule-widget-api.gs` (레포 루트) |

각 파일 저장 (💾) → **배포 → 배포 관리 → 편집 → 새 버전 → 배포** (URL 유지 위해 같은 deployment ID에 새 버전)

#### 2-2. 대시보드 API 프로젝트 (1개 파일)

| 파일명 | 새 코드 위치 |
|---|---|
| Code | `legacy-gas/dashboard-api.gs` |

저장 → 배포 → 새 버전 → 배포

---

### STEP 3. 트리거 확인 (5분)

GAS 콘솔의 트리거 메뉴(⏰)에서 현재 등록된 트리거 확인:

| 함수명 | 이벤트 | 비고 |
|---|---|---|
| `onEditTrigger` | onEdit (스프레드시트) | **유지** — 이번 버전에서 ID 자동 발급도 같이 처리 |
| `moveRowOnCheck` | onEdit (스프레드시트) | 유지 |
| `syncCompletedToDataSheet` | 시간 기반 (매일 오전 9시) | 유지 |
| `syncCompletionRate` | (있다면) | **삭제** — 함수가 제거됨 |
| `handleUrgentTask` | (이미 해제) | 유지 (해제 상태 그대로) |

---

### STEP 4. 마이그레이션 함수 3개 실행 (10분)

기존 행들에 ID 일괄 부여. **각 함수 한 번씩 실행**:

#### 4-1. 신규·유지보수 시트 → schedule-widget-api 프로젝트

GAS 콘솔 → schedule-widget-api 파일 → 함수 선택 드롭다운에서 **`widgetMigrateScheduleIds`** → ▶ 실행

로그에 `✅ 신규·유지보수 시트: N개 행에 ID 부여` 확인.

#### 4-2. 완료 시트 → 같은 프로젝트

같은 곳에서 **`widgetMigrateCompletedIds`** 선택 → ▶ 실행

로그에 `✅ 완료 시트: N개 행에 ID 부여` 확인.

#### 4-3. 데이터 시트 (8000+행) → Synccompletedtodatasheet 프로젝트

GAS 콘솔 → Synccompletedtodatasheet 파일 → **`migrateDataSheetIds`** 선택 → ▶ 실행

**주의**: 행이 많아서 GAS 6분 한도에 걸릴 수 있음. 로그 메시지 확인:

- ✅ `전체 마이그레이션 완료` → 끝남
- ⏱ `시간 한도 도달... 다시 실행하면 이어서 진행` → **같은 함수 다시 ▶ 실행 (반복)**

진행 상태는 자동 저장되므로 중단해도 다음 실행에서 이어서 진행. `전체 마이그레이션 완료` 메시지 나올 때까지 반복.

(처음부터 다시 하려면 같은 파일의 `resetMigrateDataSheetIdsProgress` 실행)

---

### STEP 5. L열 / R열 숨김 처리 (1분)

ID 컬럼은 디자이너에게 시각적 부담만 됨. 숨김 처리:

- 💛신규·유지보수 시트: L열 헤더 우클릭 → "열 숨기기"
- 💚완료 시트: L열 헤더 우클릭 → "열 숨기기"
- 업무 데이터 시트: R열 헤더 우클릭 → "열 숨기기"

---

### STEP 6. 검증 (5분)

#### 6-1. ID 자동 발급 확인

신규·유지보수 시트에서 빈 행에 임시로 광고주명 한 줄 입력 → 1~2초 후 L열(숨김 해제해서 확인)에 36자 코드 자동 등장하면 정상.

테스트 후 그 임시 행 삭제.

#### 6-2. 위젯 동작 확인 (Phase 1: v0.2.3 빌드 배포 전이면 스킵 가능)

위젯이 이미 실행 중이면 트레이 → 새로고침. GAS 응답에 id 필드가 포함되는지 확인 (위젯이 정상 동작하면 OK, 별도 디버그 필요 없음).

---

### STEP 7. 위젯 v0.2.4 빌드 + 배포

publish 토큰 해결 후:

```
npm install
npm run build:win
```

(맥에서 빌드하려면 별도 환경 셋업 필요. 윈도우 PC 또는 GitHub Actions 사용 권장)

GitHub Releases에 publish되면 5명 위젯이 다음 실행 시 자동 업데이트.

## 트러블슈팅

| 증상 | 원인 | 조치 |
|---|---|---|
| 마이그레이션 함수 실행 시 "Exceeded maximum execution time" | GAS 6분 한도 | 함수 다시 실행 (이어 진행됨) |
| 신규 행 추가해도 L열에 ID 안 들어옴 | onEditTrigger 미동작 | GAS 트리거 메뉴에서 onEditTrigger 등록 상태 확인. 없으면 트리거 추가 (Scheduler 프로젝트 → onEditTrigger → 스프레드시트 onEdit) |
| 위젯에서 "잘못된 요청" 토스트 반복 | 응답 구조 변경 | 위젯이 v0.2.3 이하 → v0.2.4 빌드 배포 필요 |
| 대시보드에서 "필수 컬럼 없음: TAT" 에러 | 데이터 시트 Q열 헤더가 아직 "소요일" | Q1 셀 헤더를 `TAT`로 변경 |
| 완료 시트 행 이관 시 잘못된 컬럼에 값 박힘 | Scheduler.gs가 구버전 | Scheduler.gs 새 코드 붙여넣기 + 저장 재확인 |
